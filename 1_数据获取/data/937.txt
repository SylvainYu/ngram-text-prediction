　　原标题：微软发布「云上太空」计划：马斯克SpaceX星链加持，让云服务遍布全球
　　微软试图在云服务领域向全球行业的领军者 AWS 发起挑战，他们最新的策略竟是：与伊隆 · 马斯克旗下的 SpaceX 合作，把云服务推向太空。
　　这几年里，我们讨论过云上贵州、乌兰察布，甚至「云上加州」，但微软本周二公布的一项新计划「Azure Space」率先把云服务带到了太空，着实领先业界一大步。
　　我们知道，自去年起，SpaceX 的「星链」StarLink 网络已经开始加速部署，其计划通过近地轨道卫星群，提供覆盖全球的高速互联网接入服务。这样一张网，是微软发展云服务的理想合作伙伴。在发布 Azure Space 时微软还表示，除了 SpaceX 以外，该公司还将与卢森堡卫星运营商 SES 展开同样的合作。
　　「将云服务推向太空的行动将是互联网的一次巨大飞跃，」微软 Azure 全球副总裁 Tom Keane 表示。「航天业界的迅速发展正在降低人们链接公众的门槛，通过 Azure Space 计划，我们可以实现农业、能源、通信和政府业务等领域的新发展。」
　　微软已经召集了由著名太空行业专家组成的团队，并与世界一流的产品工程师和科学家共同构建满足太空需求的云计算能力。创新领域包括模拟太空任务、从卫星数据中发掘有用信息以及在地面和轨道上赋能创新。
　　通过与太空发展社区的领导者合作，微软将通过遍布全球的卫星连接扩展 Azure 功能，解锁更多场景中的云计算能力，并赋能合作伙伴和客户以实现更大的目标。
　　在世界各地实现快速、安全的卫星通信网络
　　海底、陆地和地铁光纤组成的超过 16 万英里的全球网络帮助世界各地的人们实现互联互通。但对于一部分身处偏远、恶劣环境中的运营商来说，日益增长的数据和宽带访问需求很难得到满足。
　　人们越来越需要一个蓬勃发展的卫星供应商生态系统。今日，微软宣布与 SpaceX 和 SES 展开合作，作为 Azure Space 生态系统的一部分。
　　具体而言，微软将分别在以下两方面与 SpaceX 和 SES 展开合作：
　　与 SpaceX 的「星链」StarLink 网络展开新的合作，为全新的 Azure 模块化数据中心（Modular Datacenter, MDC）提供高速、低延迟的卫星宽带；
　　基于与 SES 合作的现有 Azure Orbital，支持 O3B 中地球轨道（Medium Earth Orbit, MEO）星座，扩展云数据中心区域和云边缘设备之间的连通性。

　　基于卫星的通讯除了覆盖面广之外，也有延迟更低的好处。在今年的活动中，马斯克曾表示 Starlink「网络连接的体验会非常好，现在我们的目标是将延迟降至不到 20ms。」在卫星数量足够多之后，用这个网络打竞技游戏也是非常顺畅的事。
　　如何实现基于卫星的云服务呢？微软将提供多轨道、多波段、多供应商和云化能力，带来全面的卫星互联解决方案，最终满足客户需求。
　　灵活的卫星通信加之 Azure 提供高性能计算、机器学习和数据分析的能力，这些为公共和私立机构提供了大量新的发展机遇。
　　微软在卫星通信解决方案上的合作方式有助于更快地为客户提供这些能力，有助于解决他们在关键任务中的太空需求。
　　除了近期宣布的 Azure 轨道地面站服务以及与 SES、Intelsat 和 ViaSat 建立的 ExpressRoute 卫星供应商合作伙伴关系，这些新的互联合作将提供更多的卫星能力，有助于将有价值的客户数据从地球同步轨道（GEO）卫星直接导入 Azure。
　　在挑战性任务中构建独立的数据中心

　　卫星一侧用的是 SpaceX，那么地面站是什么样子？微软表示我有现成的：很多机构正通过 Azure Stack 等产品在智能边缘充分利用云计算的优势。微软已经在极端环境中数据中心试点建设方面取得了巨大的飞跃，如著名的 Project Natick 水下数据中心研究项目。基于以往的经验以及客户最艰难任务的需求，微软 MDC 将进一步提升这种边缘计算能力。
　　在电力和基础设施建设等关键前提条件不可靠等挑战性环境中，微软 MDC 将依然支持高强度、安全的云计算。用户可以根据自身需要在独立的单元中使用 Azure。MDC 可以使得机构在偏远区域部署完整的数据中心，或者利用 field-transportable 解决方案来增强现有基础设施。MDC 主要在地面光纤和低宽带网络上运行，也可以完全断开连接。
　　通过卫星合作伙伴，卫星连通附加组件使得 MDC 客户更灵活地访问 Azure 的超大规模基础服务。
　　Azure 如何助力航天任务
　　随着航天任务和卫星功能的日益普及，微软也在太空数字技术方面不断探索。其中第一个是 Azure 轨道模拟器，许多商业机构和政府组织正在开发数千个互联卫星星座，这些星座需要进行精确的规划和复杂的 AI 驱动的编队协议，以确保网络连接良好和轨道运行正常。
　　Azure 轨道模拟器可以在进行大规模的卫星星座仿真，这样就能够使得卫星开发人员在发射单个卫星之前，对卫星网络和 AI 算法进行训练。Azure 可以模拟整个卫星网络，包括使用虚拟或实体卫星硬件直接处理后的预收集卫星图像做复杂的实时场景生成。
　　SpaceX 总裁兼运营总监 Gwynne Shotwell 在接受采访时表示，微软是与 SpaceX 合作进行 SDA 计划的分包商。
　　「我们将向政府交付许多卫星，这些卫星具有防御弹道武器的能力。」
　　「微软作为分包商将承担很多工作，这对于我们的关系来说是一种有趣的转折。」
　　据介绍，这些新功能为已经为 Azure 的太空合作伙伴机构带来了新的机遇，比如：
　　美国国防创新单元（DIU）选择了微软和 Ball Aerospace 来创建更具实用性的太空数据，重新构想地面站技术，搭建了一个具备快捷云处理能力的解决方案，以支持美国空军的 CASINO（Commercially Augmented Space Inter Networked Operation） 项目。
　　猎户座（Orion）宇宙飞船的总承包商洛克希德 · 马丁已经将 HoloLens 2 用于航天器的组装任务，支持 NASA 将人类送到月球的 Artemis 计划。
　　空客公司使用 Microsoft Azure Stack 的敏捷性、创新性和竞争优势来搭建其航空解决方案。
　　环境地质公司 Seequent 利用卫星数据和 Azure 计算能力助力其全球范围内的水质、水量等勘察工作，以便能够清晰地查看地下水和污染物，所形成的软件方案应用于数百个项目之中。
　　Land O’Lakes 是最大的农民合作组织之一，它使用 Azure FarmBeat ，利用卫星图像以及来自传感器、无人机、农用设备的数据来实现数字农业。利用来自卫星的光学和多谱段图像的 AI 模型有助于判断作物长势、预测其产量等。
　　微软认为，Azure Space 有助于解决客户在太空中面临的一些严峻技术挑战：处理卫星产生的大量数据，将云服务和宽带送到最偏远的地域，以及设计高度复杂的太空系统。微软表示，随时准备好为客户在环地球或地球之外的太空飞行任务提供支持，并利用云和太空的力量来帮助各行各业重新设计方案，以解决世界上最具挑战性的问题。
　　从地球竞争到太空
　　对于云服务巨头来说，把战火烧到太空中并不是心血来潮。近几个月来，微软已经在太空环境中测试了 Azure 云服务的能力，其服务在美国监管文件中显示，经营范围包括「灾害的预测和跟踪，提高供应链和经济活动的可见性等方面」。
　　微软的「云上卫星」仅比亚马逊的太空云服务计划提前了大约 3 个月。今年六月，亚马逊成立了专门的业务部门，面向太空相关云服务的计划。亚马逊已将 Maxar 和 Capella Space 视为客户，这些伙伴可以帮助它管理来自卫星的数据。
　　除了合作之外，亚马逊创始人贝索斯旗下的太空公司蓝色起源（Blue Origin）正计划在 2021 年首次发射自己的 New Glenn 火箭，其还计划部署一套与 SpaceX 的「星链」直接竞争的 Project Kuiper 卫星通讯系统，如最终部署完成，后者将包含多达 3236 颗卫星。
　　亚马逊有义务在 2026 年以前发射三千多颗卫星数量的至少一半，从而保留 FCC 授予该公司的运营许可证。亚马逊表示将投资 100 亿美元来打造这个星座，其中的卫星与星链相同，将在近地轨道（LEO）上运行。
　　相对的，「星链」计划在 2024 年底之前部署 12000 颗卫星。

　　在未来，卫星星座将成为 AWS 庞大生态系统的一部分。「对于亚马逊来说，这是完全不同的局面，」斯坦福大学航空航天学助理教授 Zac Manchester 表示。「让亚马逊与 SpaceX、OneWeb 有所不同的是，前者还有很多其他领域的业务。」如果 Kuiper 成功，那么亚马逊不仅可以提供全球范围的卫星宽带，还可以把网络服务作为 AWS 服务的一部分，而 AWS 已经提供了包括云计算、机器学习、大数据分析等各种能力。
　　出于成本考虑，基于卫星通讯的云服务最初的客户可能会来自国防、石油天然气、航空等领域，这些行业需要传输、存储和处理大量数据。
　　一些分析师认为，在 2030 年左右，与太空相关的云服务产业总收入可能会达到 150 亿美元。
